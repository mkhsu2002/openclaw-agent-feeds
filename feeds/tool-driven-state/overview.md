# ⚙️ [核心升級] 工具驅動防護決策迴圈 (Tool-Driven Decision Loop)

### 📄 檔案簡介
本飼料包專為**硬核開發者**設計。它將 LLM 從一個「對話者」轉變為一個**嚴格的邏輯執行器**。代理的每一步行動都由工具回傳的「狀態值」決定，而非其自身的意志。

### 🚀 升級核心
1. **狀態轉移矩陣 (State Transition Matrix)**：定義具體的「狀態」與「目標」，只有滿足條件才能切換。
2. **工具回傳依賴**：代理禁止在未調用工具的情況下預測結果。
3. **重試與終止機制**：當狀態陷入循環或錯誤時，具備標準化的自動恢復路徑。

### 📋 建議模型
- **推薦版本**：GPT-5.3 / Gemini 3.0 Pro / Claude Sonnet 4.6 (優秀的 Function Calling 穩定性)
- **最低版本**：GPT-5.1 / Gemini 3.1 Flash / Claude Haiku 4.5 (2026 Gen)
